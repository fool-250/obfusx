ObfusX - 强大且轻量的 HTML 防护混淆器

ObfusX 是一款 零依赖、纯前端 的网页混淆防护工具，由原生 JavaScript 实现，无后端、无框架。

在不影响用户交互 的前提下，ObfusX 通过对结构层混淆和行为层的防护，能有效提升网页内容的 抗复制、抗调试、抗抓取 能力，为你的原创页面提供强力保护。

📑 混淆功能说明

🔹 默认选项

说明：会默认启用清除注释、压缩。并随机使用3~6种混淆，比如：控制流重构、回调函数包装、分步调度注入、分支延迟执行、RC4 字符串编码、全局变量重命名、反调试、自我防御保护和对象密钥混淆等等。

效果：适用于大多数网页内容的快速保护。

🟡 注意：默认混淆不会隐藏 DOM 结构，仅封装源码。若需进一步的防护，需要结合其他功能选项。

🔹 文本拆解伪装

说明：将 <body> 内的所有静态文本拆分为多个自定义节点 <obfusx>，避免真实文本出现在 DOM 中。

效果：让复制者抓不到重点，让爬虫只能抓到噪声，提升文本内容防护强度。

🔹 随机属性注入

说明：在 DOM 元素中插入随机生成的属性名与伪类名，制造结构噪声，增加样式与脚本的定位难度。

效果：干扰选择器定位与样式分析，提升混淆复杂度。

🔹 标签语义变异

说明：将标准标签（如 <div>）替换为无语义标签（如 <obfusx-com>）。

效果：打乱语义结构，干扰自动化识别与样式匹配。

🔹 ID 替换变异

说明：使用哈希或随机字符串替换所有静态 ID。

效果：防止通过 ID 快速定位关键元素，提升结构防护。

🔹 隐形签名

说明：在 DOM 中注入不可见的自定义签名内容，携带唯一签名标识，可通过代码查询验证。

效果：用于原创内容的追踪与归属验证。

查询方法：复制下列代码在控制台查询

getComputedStyle(document.documentElement).getPropertyValue("--obfusx-signature")

// 你的签名会显示在这里

🔹 链接伪装

说明：通过修改 <a> 标签的 href、rel、target 等属性，或嵌套无效子元素，干扰链接的语义识别与爬虫解析。

效果：防止链接被直接识别、抓取或镜像。

🔹 假元素

说明：在关键节点附近插入无功能的 DOM 元素，赋予干扰性 ID 或类名，模拟真实结构。

效果：增加页面节点数量，干扰结构解析与调试工具。

🔹 锚点干扰

说明：注入空锚点与伪造 ID，干扰浏览器定位与爬虫跳转逻辑。

效果：打乱页面定位逻辑，干扰自动跳转与爬虫锚点识别。

🔹 模块化

说明：将脚本逻辑拆分为多个函数模块，并打乱执行顺序与命名。

效果：提升代码不可逆性，增加分析难度。

🔹 字符扰乱

说明：在文本节点中插入零宽字符（如 \u200B、\u200C），制造语义差异，干扰解析与匹配。

效果：阻碍复制、粘贴与选择器定位，提升内容的抗提取性与防抓取能力。

🟡 注意：启用“文本拆解伪装”时，该功能将自动关闭以避免冲突。

查询方法：复制下列代码在控制台查询

const node = document.querySelector('body'); // 或其他包含静态文本的元素 如 <html>
if (node) {
  const codes = [...node.textContent].map(c => c.charCodeAt(0));
  console.log("字符码位:", codes);
  console.log("字符个数:", codes.length);

  const hasGhost = codes.filter(c => c === 8291).length;
  console.log("幽灵字符数量:", hasGhost);
}

// 插入的字符会显示在这里

🔹 快捷键

说明：监听键盘事件并阻止默认行为，屏蔽一些常用基本快捷键。

效果：有效阻止非技术用户快速进入开发者工具或克隆你的内容。

🔹 防嵌套访问保护

说明：检测 window.top !== window.self 并强制跳转或阻止渲染，防止 iframe 嵌套。

效果：防止盗链、嵌套调用与恶意嵌入。

🔹 干扰开发者工具

说明：ObfusX 会通过检测 RegExp、Date 和 Function 等原生对象的 toString() 方法是否被代理或篡改，结合 performance.now() 判断是否被调试器挂起，并监听窗口判断 outerWidth - innerWidth 是否超过阈值等方法，尝试检测并干扰开发者工具的使用。

效果：在绝大多数浏览器环境中，能有效中断调试流程，清空页面内容，或触发跳转。即使页面未被关闭，也只能看到由 ObfusX 混淆后的代码。

🔹 下载防护

说明：通过 CSP 或动态加载方式阻止外部 .css / .js 文件被直接保存。

效果：使 游览器另存为 无法直接下载你的文件，保护你的外部静态文件。

🔹 访问速率限制

说明：记录访问时间戳与频率，超出阈值后触发封锁逻辑。

效果：防止高频爬虫或恶意刷新行为。

🔹 域名锁定并重定向

说明：检测当前访问域名是否在白名单中，非法域名触发跳转或阻止渲染。

效果：防止非法镜像、盗链与跨域调用。

🔹 水印保护

说明：通过 CSS 背景图层或伪元素插入水印文字。

效果：提升内容归属感，增强视觉防复制效果。

❓常见问题解答（FAQ）

ObfusX 是否会存储我的源码？

不会。ObfusX 是纯前端工具，所有处理过程都在你的浏览器本地完成，不会上传或存储你的源码。

混淆后是否就绝对安全了？

不是。任何在浏览器中运行的 JavaScript 都必须被解析执行，不存在”绝对安全“。ObfusX 可以显著提升源码的逆向难度，增加安全性。

为什么混淆后的代码比原始源代码大？

ObfusX 使用多种不同的混淆方式保护你的代码，混淆后的代码体积肯定会增加，这是正常现象。

混淆是否影响 SEO？

文本拆解伪装、标签语义变异、链接伪装、字符扰乱、锚点干扰这几个混淆项会影响爬虫解析，建议根据实际需求选择。

ObfusX 能混淆我的 JS 文件吗？

不能。ObfusX 专注于 HTML 页面级混淆，所有处理逻辑都围绕完整网页结构展开。不会对独立 JS 文件进行混淆。建议先使用如 javascript-obfuscator 等工具处理脚本，再将完整页面交由 ObfusX 混淆。

混淆后能还原代码吗？

ObfusX 的混淆是不可逆的，无法自动还原为原始源码。请在混淆前备份原始文件。
